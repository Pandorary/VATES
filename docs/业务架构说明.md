# Vates - 业务架构说明

## 1. 产品定位

**Vates**（交易状态感知与风控辅助系统）是面向 A 股个人投资者的 Web 端决策辅助工具。核心价值：帮助投资者感知市场环境、识别经典交易模式、量化评估盈亏比，并通过每日复盘形成交易纪律。

**核心原则**：仅展示状态与概率，不提供任何买卖建议。

---

## 2. 用户角色

| 角色 | 说明 |
|------|------|
| 普通用户 | A 股个人投资者，使用系统查看市场状态、自选股模式匹配、使用盈亏比计算器 |
| 管理员 | 系统运维，管理市场配置参数、监控数据抓取任务（MVP 暂不涉及后台界面） |

---

## 3. 核心业务域

### 3.1 市场环境感知域

**业务目标**：让用户一眼看懂当前市场水温，决定仓位轻重。

**核心实体**：
- 市场情绪（ICE 冰点 / WARM 回暖 / HIGH 高潮 / RETREAT 退潮）
- 资金攻击方向（主力净流入 Top 板块）

**业务规则**：
- 每天盘后（15:40）重新计算市场状态
- 状态判定基于四个量化指标：最高连板高度、晋级率、炸板率、昨日涨停今日平均收益
- 阈值可通过 `market_config` 表动态调整，无需改代码

**用户场景**：
1. 用户打开首页，看到大卡片显示当前市场温度（色块 + 关键数值）
2. 下方展示今日主力资金攻击的板块方向
3. 用户据此判断当日是否适合交易、仓位比例

### 3.2 经典模式识别域

**业务目标**：自动扫描个股是否出现历史上高胜率的交易模式。

**核心实体**：
- 交易模式（如"平台突破放量"）
- 模式信号（某股某日触发了某模式）
- 模式条件（量比、价格位置、资金流入等）

**业务规则**：
- 模式由一组可配置的条件规则定义，返回 True/False
- 扫描全市场（或用户自选股），每日盘后执行
- 每个模式信号附带：确认条件、失败条件、风控参考位
- 必须展示历史统计（胜率、盈亏比），不能只展示收益

**用户场景**：
1. 用户搜索某只股票，进入个股详情页
2. 页面展示该股当前匹配了哪些经典模式
3. 每个模式卡片显示：触发条件现状、确认/失败条件、历史胜率、风控位
4. 若无匹配，显示"当前未触发任何经典模式"

### 3.3 盈亏比评估域

**业务目标**：帮助用户在入场前量化评估风险收益比。

**核心实体**：
- 入场价、止损价、止盈价

**业务规则**：
- 盈亏比 < 2 显示警告（不值得交易）
- 盈亏比 >= 3 显示绿色标记（理想交易）
- 纯前端计算，无需后端

**用户场景**：
1. 用户在个股 K 线图上拖拽三条水平线（入场/止损/止盈）
2. 侧方面板实时显示：潜在亏损比例、潜在盈利比例、盈亏比
3. 根据盈亏比数值显示不同颜色标记

### 3.4 自选与观察域

**业务目标**：管理用户关注的股票，监控模式信号变化。

**核心实体**：
- 自选股列表
- 观察池（触发了模式边界条件的个股）

**业务规则**：
- 自选股列表展示当前匹配模式（最重要列）
- 观察池独立 Tab，存放待确认个股
- 支持从自选移入观察池，反之亦然

**用户场景**：
1. 用户添加关注的股票到自选列表
2. 列表显示每只股票当前是否有模式匹配
3. 触发边界条件的股票自动进入观察池
4. 用户确认后移回自选或删除

### 3.5 复盘简报域

**业务目标**：每日自动生成交易复盘，形成纪律性回顾习惯。

**核心实体**：
- 每日复盘报告（市场状态 + 自选信号 + 风险提醒）

**业务规则**：
- 每日盘后 17:10 自动生成
- 支持回溯历史任意交易日
- 内容结构：今日战况、模式信号、风险预警

**用户场景**：
1. 用户打开复盘页，默认显示最近交易日简报
2. 通过日期选择器回溯历史某天的复盘
3. 简报清晰展示当天市场状态、自选股触发的信号、风险提醒

---

## 4. 业务边界（不做什么）

- 不提供买卖建议（买/卖/持有）
- 不展示预期收益率承诺
- 不自动下单交易
- 不提供实时行情推送（MVP 阶段轮询）
- 不保证盈利

---

## 5. 合规约束（强制）

- 所有页面底部标注："仅供参考，不构成投资建议"
- 首次访问显示全屏免责弹窗，需同意方可进入
- 模式展示必须同时呈现成功与失败案例
- 数据来源必须为合法公开 API
