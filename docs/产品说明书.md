

## AI股票预测与持仓诊断系统 产品说明书

**版本**：V1.1  
**最后更新**：2026年5月30日

---

## 一、产品概述

### 1.1 产品定位

本系统是面向个人投资者的**AI辅助投研工具**。产品以“让AI自动完成数据获取、信息提炼、逻辑推演、预测输出”为核心理念，依托大语言模型的文本理解与推理能力，结合工程化的数据真实性保障机制，实现标的智能识别、个股走势预测、行业研判、自动化跟踪复盘、持仓诊断分析，以及后台提示词的统一管控。

核心围绕**「数据获取→信息萃取→可解释性预测→跟踪复盘→偏差归因→持续迭代」**的全链路闭环设计。操作路径极简，全流程自动化的同时，确保每个预测结论有据可查、有源可溯、有法可证伪。

### 1.2 核心能力

1. **智能搜索分类**：自动识别输入内容为个股、行业或未知，精准路由至对应分析流程。
2. **多源数据自动抓取与交叉验证**：系统自动从多个权威财经信源获取数据，交叉验证后生成置信度标签。
3. **可解释性AI预测**：个股支持四个固定周期的走势预测；行业输出一份包含短期（1-3个月）和长期（半年以上）双时间维度的研判报告。每个预测均包含完整推理链、多空逻辑罗列、置信度评估及失效条件。
4. **自动化跟踪复盘与偏差归因**：按预设周期自动校验预测结果，分析偏差产生的数据原因或逻辑原因，生成可追溯的复盘记录。
5. **持仓智能诊断**：对用户持仓个股做研判分析，输出操作建议并支持复盘记录。
6. **后台提示词统一管理**：全链路所有大模型调用节点均配置独立提示词模板，支持激活、停用、版本管理，统一管控AI输出规范。

### 1.3 适用人群

个人股票投资者，尤其是有一定交易经验的短线/中线交易者。

## 二、整体业务流程总览

1. 用户首页搜索 → 系统调用大模型进行**智能分类**（个股/行业/未知）。
2. 分类确认为有效标的后，系统自动触发**多源数据抓取与交叉验证**管道，获取并校验标的的近期行情与基本面数据。
3. 校验通过的数据注入对应的**预测提示词模板**，大模型生成包含推理链与置信度评估的预测报告。
   - **个股**：用户选择预测时段后生成预测报告。
   - **行业**：直接生成包含短期与长期研判的综合报告。
4. 用户查看预测报告后，可手动点击**“保存预测记录并开启跟踪”**，系统正式建立预测档案并绑定对应复盘模板。
5. 系统按预设周期自动执行**跟踪复盘**（调用配套复盘提示词），比对预测与真实走势，输出偏差归因分析。
6. 用户可在历史记录中查看全量预测档案、跟踪日志与复盘报告。
7. **持仓模块**独立运行，调用专属提示词完成诊断与复盘。
8. **后台管理端**统一维护所有场景的提示词模板，支持即时生效与版本追溯。

## 三、智能数据引擎与可解释性预测系统（核心中枢）

本模块是连接“用户搜索”与“AI预测”的核心管道，是产品可落地、可信任的基础设施。它负责自动获取、清洗、交叉验证数据，并确保所有预测结论均基于明确可追溯的事实链。

### 3.1 设计原则

- **不创造数据，只搬运可验证的公开数据**。
- **宁可终止预测，也不基于不可信数据生成结论**。
- **所有用于预测的数据点，均可追溯到原始来源与抓取时刻**。

### 3.2 数据获取机制：多源定向抓取

系统根据标的类型，自动生成带严格限制的搜索请求，从至少2-3个不同权威信源获取数据。

**权威信源白名单**：

- 财经网站行情页：东方财富、新浪财经（强制指定域名）
- 官方公告源：交易所官网、巨潮资讯网
- 数据服务商：如已接入API，优先使用；否则使用公开页面抓取

**搜索约束规则**：

- 自动拼入精确交易日期，杜绝历史数据污染。
- 强制限定权威域名，过滤自媒体、论坛等非结构化噪音源。
- 搜索结果缓存原始页面内容，作为后续审计的底稿。

### 3.3 数据提取与交叉验证机制

系统使用专用的 **【数据提取与校验】提示词模板**，对抓取到的原始文本进行结构化提取。该提示词要求大模型以**提取器**身份工作，严禁在此步骤进行任何预测或分析。

**提取的数据字段（个股）**：
| 字段 | 说明 |
|------|------|
| `latest_close_price` | 前一交易日收盘价 |
| `open_price` | 前一交易日开盘价 |
| `high_price` | 前一交易日最高价 |
| `low_price` | 前一交易日最低价 |
| `price_change_percent` | 涨跌幅 |
| `volume` | 成交量 |
| `key_events` | 近3日重要公告或新闻标题列表 |
| `data_sources` | 各字段的数据来源URL |

**提取的数据字段（行业）**：
| 字段 | 说明 |
|------|------|
| `sector_index` | 行业指数最新点位与涨跌幅 |
| `leading_stocks_performance` | 龙头个股近期表现摘要 |
| `policy_news` | 近期行业政策与重大事件 |
| `fund_flow` | 板块资金流向概况 |
| `data_sources` | 数据来源URL |

**交叉验证规则**：

1. **多数投票**：同一字段从不同信源提取后，采用多数一致值。若仅两个信源且冲突，标记为冲突。
2. **合理性校验**：对可计算字段进行反向验证，不一致即标记`data_conflict: true`。
3. **强制时间戳校验**：验证所提取数据对应的交易日期是否与目标日期一致。

### 3.4 数据置信度分类

所有提取到的数据，均按以下标准标注置信度标签：

| 置信度      | 判定标准                   | 预测流程行为                   |
| -------- | ---------------------- | ------------------------ |
| **高**    | ≥2个权威信源数据完全一致，且通过合理性校验 | 正常生成预测报告                 |
| **中**    | 仅1个权威源成功提取，或少数字段缺失     | 仍可生成预测，但UI显著提示数据来源单一     |
| **低/冲突** | 多源数据严重不一致，或核心字段缺失      | **系统拒绝生成预测**，展示信息摘要供用户参考 |

### 3.5 可解释性预测输出规范

当数据置信度为“高”或“中”时，系统将结构化数据注入对应预测提示词，生成预测报告。**所有预测报告必须包含以下五个标准模块：**

**模块一：核心结论**
一句话概括预测方向，如“短期看多，但波动可能加剧”。

**模块二：数据依据**
列出支撑该结论的2-3条关键萃取事实，并标明数据来源。

**模块三：多空逻辑罗列**
分别列出看多理由与看空风险，展示完整的权衡过程。

**模块四：置信度评估**

- **数据置信度**：高/中
- **预测置信度**：高/中/低（由大模型基于信号一致性、市场不确定性等因素评估）

**模块五：失效条件与关键观察位**
明确说明什么情况下该预测不再成立。

### 3.6 数据溯源与审计

每条预测记录在数据库中均包含一个完整的数据快照：提取后的结构化JSON数据、各信源的原始URL列表、抓取时间戳、置信度标签。复盘时，系统直接读取此快照进行比对，无需二次抓取，确保复盘结果精确可验证。

## 四、前端功能模块详细说明

### 4.1 首页-智能搜索模块

#### 4.1.1 功能描述

系统入口核心模块。用户输入文本后，系统自动完成内容分类与标的确认，分流至对应预测流程。

#### 4.1.2 操作流程

1. 用户在搜索框输入文本（个股名称/代码、行业名称等）。
2. 系统调用 **【内容智能分类】提示词**，判定类型：个股/行业/未知。
3. **分类结果处理**：
   - **个股**：在输入框下方展示确认后的标的名称及代码，进入**时段选择**环节。
   - **行业**：直接确认行业名称，跳过时段选择，自动触发行业预测生成。
   - **未知**：弹出引导提示，建议输入标准个股代码、个股名称或规范行业名称。
4. 如有个股多个可能匹配项（如“长安”对应多只个股），系统展示候选列表供用户点选确认，获取唯一代码后继续。

#### 4.1.3 预测时段选择（仅个股）

个股提供**四类固定预测周期**，用户单选其一：

- 下一交易日预测
- 未来5个交易日预测
- 未来1个月预测（约20个交易日）
- 未来3个月预测（约60个交易日）

行业预测**无时段选择**，直接生成一份包含短期与长期两个维度的研判报告（详见4.2.2）。

### 4.2 AI预测与跟踪核心流程

#### 4.2.1 个股预测生成规则

根据用户所选时段，系统自动匹配对应专属提示词，结合数据引擎输出的结构化数据生成预测报告：

| 时段      | 调用提示词模板    |
| ------- | ---------- |
| 下一交易日   | 个股预测-下一交易日 |
| 未来5个交易日 | 个股预测-一周    |
| 未来1个月   | 个股预测-1个月   |
| 未来3个月   | 个股预测-3个月   |

#### 4.2.2 行业预测生成规则

行业预测调用 **【行业研判】提示词**，基于数据引擎提取的行业数据，生成一份**双时间维度研判报告**：

- **短期研判（1-3个月）**：分析当前行业景气度、资金面、短期催化因素，给出短期走势判断与关键观察变量。
- **长期研判（半年以上）**：分析产业趋势、政策方向、基本面变化，给出中长期配置价值判断与核心逻辑。

行业预测报告同样须包含第三节所述的五个标准模块，其中核心结论需对短期和长期分别表述。

#### 4.2.3 保存与开启跟踪规则

1. 用户查看预测报告后，**必须手动点击“保存预测记录并开启跟踪”**，系统才正式建立预测档案并绑定复盘模板。
2. 若用户不执行此操作：仅临时展示结果，不保存、不跟踪、不生成复盘日志。
3. 确认后，系统将当前预测的数据快照、预测结论、时间戳一并存入数据库。

#### 4.2.4 自动化跟踪与复盘规则

跟踪复盘均调用该预测绑定的对应复盘提示词，执行周期固定：

**个股跟踪复盘周期**：

| 预测周期    | 复盘时机       | 复盘内容                                  |
| ------- | ---------- | ------------------------------------- |
| 下一交易日   | 下一交易日收盘后   | 比对预测方向与实际走势，计算偏差，归因分析（数据问题/逻辑问题/突发事件） |
| 未来5个交易日 | 5个交易日全部结束后 | 逐日跟踪日志汇总，整体方向性复盘，分析中间波动               |
| 未来1个月   | 月末一次       | 趋势性复盘，分析期间重大事件对走势的影响                  |
| 未来3个月   | 每月一次+期末终评  | 阶段复盘，重点分析政策、黑天鹅等长周期因素                 |

**行业跟踪复盘周期**：
行业预测包含短期和长期两个时间窗口，系统分别执行复盘：

- **短期复盘**：自预测保存日起1-3个月窗口期满后执行，依据期间行业指数实际表现，评判短期研判的准确性。
- **长期复盘**：自预测保存日起6个月后执行，评判长期趋势判断的合理性。
- 两次复盘均调用 **【行业复盘校验】提示词**，生成独立的复盘报告，并关联到同一条行业预测记录。

**复盘输出内容**：

- 预测准确度评级（准确/部分准确/偏差较大/完全错误）
- 偏差原因归类（数据源问题/逻辑推理缺陷/不可预见事件）
- 调整建议
- 复盘记录自动归档至对应预测档案

#### 4.2.5 预测记录管理

1. **记录分类**：分为“个股预测记录”和“行业预测记录”两大列表。
2. **存储数量限制**：
   - 个股预测记录：最大存储 **10条**。
   - 行业预测记录：最大存储 **3条**。
3. **超限处理**：达到上限时弹出提示“预测记录已达存储上限，请删除无用记录后再新增”。
4. **记录详情**：点击标的名称展开详情页，包含：
   - 原始预测报告全文
   - 数据快照（可查看预测时的数据依据及来源）
   - 全量跟踪日志
   - 复盘报告（行业预测将有短期和长期两份复盘报告）
   - 调整记录
5. **状态标识**：列表中以标签区分“跟踪中”、“已复盘-符合”、“已复盘-偏离”、“已失效”等状态。

### 4.3 持仓诊断模块

#### 4.3.1 功能描述

针对用户持仓个股，提供AI智能诊断、情景分析、操作建议，并支持持仓复盘与记录留存。

#### 4.3.2 调用规则

- 持仓诊断：调用 **【持仓智能诊断】提示词**
- 持仓复盘：调用 **【持仓复盘校验】提示词**

#### 4.3.3 核心功能

1. **单只持仓股诊断**：
   - 现状分析（成本、盈亏、技术位）
   - 多情景推演（上涨/震荡/下跌情景分别分析）
   - 操作建议（加仓/持有/减仓/止损的逻辑说明）
   - 风险提示
2. **持仓复盘**：自动对比前期诊断结论与实际走势，记录匹配度与偏差原因。
3. **复盘记录永久留存**，支持历史查阅。
4. **联动能力**：持仓诊断详情页可一键跳转至该个股的独立预测记录页面。

## 五、后台-提示词管理模块（运维专用）

### 5.1 模块定位

仅后台管理员可见与操作，普通用户无任何访问权限。负责全链路大模型调用模板的统一配置、维护与版本管理。

### 5.2 全场景提示词分类

系统所有调用大模型的节点，均配置独立场景模板：

| 场景分类 | 模板列表                                        |
| ---- | ------------------------------------------- |
| 内容分类 | 内容智能分类                                      |
| 数据引擎 | 数据提取与校验                                     |
| 个股预测 | 个股预测-下一交易日、个股预测-一周、个股预测-1个月、个股预测-3个月        |
| 行业研判 | 行业研判（双时间维度）                                 |
| 跟踪复盘 | 个股复盘-下一交易日、个股复盘-一周、个股复盘-1个月、个股复盘-3个月、行业复盘校验 |
| 持仓相关 | 持仓智能诊断、持仓复盘校验                               |

### 5.3 模板基础字段

每条提示词模板包含：

- **模板名称**：自定义，便于识别
- **所属场景**：绑定上述固定业务场景
- **提示词内容**：大模型调用的完整Prompt文本
- **激活状态**：启用/停用
- **更新时间**：最后一次编辑时间
- **操作项**：新增、编辑、复制、停用、删除

### 5.4 核心管理规则

1. **唯一激活规则**：同一业务场景下，仅允许1条模板为激活状态。新模板激活时，系统自动将同场景旧模板设为停用。
2. **即时生效规则**：模板编辑或切换激活后，下一次调用即时使用新模板，无需重启服务或等待。
3. **删除限制**：激活状态的模板不可删除，需先停用再删除。
4. **复制功能**：支持一键复制，快速创建新版本。
5. **筛选功能**：可按业务场景、激活状态筛选列表。

### 5.5 异常兜底规则

当某一业务场景无任何已激活模板时，前端调用大模型会触发统一提示：“当前AI模板未配置，请联系管理员”，并终止本次AI请求。

### 5.6 调用日志追溯

系统自动记录每次大模型调用对应的：

- 模板ID与版本
- 调用时间
- 输入数据摘要
- 输出内容摘要
- 数据置信度标签

用于问题排查、效果评估与提示词迭代优化。

## 六、全局通用规则与约束

### 6.1 操作路径约束

1. 核心流程全程减少页面跳转，搜索、分析、预测在同一页面完成。
2. 跟踪复盘以系统自动执行为主，用户可主动查阅记录但无需手动触发。

### 6.2 数据存储约束

1. 个股预测记录：最大存储 **10条**。
2. 行业预测记录：最大存储 **3条**。
3. 持仓诊断及复盘记录：永久留存，不限数量。
4. 超限后需手动清理旧记录方可新增。

### 6.3 权限约束

1. 提示词管理模块：仅后台运维管理员操作。
2. 预测/跟踪/持仓记录：仅创建者本人可查看、删除。

### 6.4 时间口径统一

- **个股时段**：
  - 下一交易日：最近一个未来交易日
  - 未来5个交易日：从次日起连续5个交易日
  - 未来1个月：约20个交易日
  - 未来3个月：约60个交易日
- **行业时段**：
  - 短期：1-3个自然月
  - 长期：6个自然月以上
- 系统内置交易日历，自动跳过周末及法定节假日。

### 6.5 免责与合规要求

1. 所有预测报告页面底部固定展示醒目的免责声明：“本内容由AI生成，仅供参考，不构成任何投资建议。投资有风险，入市需谨慎。”
2. 预测结论不使用“必将”、“保证”等绝对化用语，不使用具体买卖指令措辞。
3. 系统不提供任何形式的投资组合管理或资金分配建议。

## 七、异常场景处理

| 异常场景           | 处理方式                              |
| -------------- | --------------------------------- |

| 预测记录达存储上限      | 提示：“预测记录已达存储上限，请清理无用记录后再新增”       |

## 八、模块联动关系总览

| 用户动作        | 系统调用链                                   |
| ----------- | --------------------------------------- |
| 首页搜索        | 调用【内容智能分类】提示词                           |
| 搜索确认为个股     | 进入时段选择，选定后触发数据抓取+对应个股预测提示词              |
| 搜索确认为行业     | 直接触发数据抓取+【行业研判】提示词，生成双时间维度报告            |
| 点击“保存并开启跟踪” | 绑定对应复盘提示词（个股按时段，行业按双窗口），存储数据快照，启动定时跟踪任务 |
| 复盘周期到达      | 系统读取数据快照与预测结论，调用复盘提示词，输出归因分析            |
| 使用持仓诊断      | 调用【持仓智能诊断】提示词                           |
| 使用持仓复盘      | 调用【持仓复盘校验】提示词                           |
| 后台模板变更      | 即时生效，下一次调用自动使用新模板                       |
