
# VATES 产品说明书 v2.0 — AI-Native 路线图

> 2026 年 5 月 15 日 | 基于项目当前代码的完整盘点与 AI 化改造路径

---

## 系统架构总览

```
VATES (Vibration-Aware Trading & Early-warning System)
├── 前端 (Vue 3 + TypeScript + Vite)
│   ├── 5 个页面：首页 / 大盘 / 自选 / 个股详情 / 复盘
│   ├── 6 个组件：AppLayout / KlineChart / PatternCard / RiskRewardPanel / Disclaimer
│   └── 1 个 store：app (免责声明状态)
│
├── 后端 (FastAPI + SQLAlchemy async + APScheduler)
│   ├── 6 个路由：market / stock / pattern / user / review / backtest
│   ├── 2 个服务：market_temp (温度计算) / pattern_scan (模式扫描)
│   ├── 1 个回测引擎：backtest (滑动窗口 + 止盈止损模拟)
│   ├── 1 个定时任务：daily_jobs (每工作日 15:40 盘后同步)
│   └── 4 组模型：stock / market / pattern / user (共 11 张表)
│
├── 数据源 (全免费、全公开)
│   ├── 东方财富 push2ex API：涨停池 / 炸板池 → 市场温度
│   ├── AkShare (新浪源)：个股日K / 板块排名 / 资金流向
│   └── 数据库 SQLite：本地缓存所有行情数据
│
└── 当前运行方式
    ├── 后端启动：uvicorn app.main:app (端口 8001)
    ├── 前端启动：npm run dev (端口 5173, 代理到 8001)
    └── 定时同步：APScheduler 进程内调度，无需额外进程
```

## 页面一：首页 `/`

### 当前能力
- **品牌展示**：VATES logo + "see the future" 标语，Google 式极简居中布局
- **股票搜索**：输入代码或名称 → 模糊匹配 → 点击跳转个股详情页

### 数据来源
| 数据 | 来源 | 逻辑 |
|------|------|------|
| 搜索结果 | 数据库 `stocks` 表 → AkShare `stock_info_a_code_name()` → mock 降级 | 三级降级链 |
| 搜索结果中的实时行情 | 数据库 `daily_quotes` 表（最新一条） | 若 DB 无数据则返回 0 |

### 能力展现方式
- 一个 24px 圆角搜索框，带 SVG 放大镜图标
- 下拉结果列表：股票名 + 代码 + 涨跌幅（红涨绿跌）

### AI-Native 改造方向
- **自然语言搜索**：用户输入"最近放量突破的票" → LLM 解析意图 → 翻译成 API 调用序列 → 返回结果
- **语义匹配**：输入"跟茅台走势差不多的" → embedding 向量相似度检索
- **对话式交互入口**：搜索框变成对话入口，逐步演进为 AI 交易助手

## 页面二：大盘行情 `/market`

### 当前能力
- **市场温度计**：ICE / WARM / HIGH / RETREAT 四个状态，渐变色彩卡片
- **四项核心指标展示**：连板高度 / 晋级率 / 炸板率 / 昨涨停均收
- **板块涨幅排行**：Top 5 板块名称 + 涨跌幅 + 领涨股

### 数据来源
| 数据 | 来源 | 逻辑 |
|------|------|------|
| 连板高度 | 东方财富 `getTopicZTPool` | 取当日涨停池中 `lbc` 字段最大值 |
| 晋级率 | 东方财富 | 昨日涨停池 ∩ 今日涨停池 / 昨日涨停池 |
| 炸板率 | 东方财富 `getTopicZBPool` | 炸板池数量 / (涨停池 + 炸板池) |
| 昨涨停均收 | 东方财富 `zdp` 字段 | 昨日涨停股今日涨跌幅均值 (采样前 50 只) |
| 板块涨幅 | AkShare `stock_sector_spot` | 按涨跌幅降序取 Top 5，GBK 编码修复 |
| 全市场资金 | AkShare `stock_market_fund_flow` | 主力净流入金额转亿 |
| 市场状态判断 | `market_temp.py` 的 `calculate()` 函数 | 硬编码阈值规则引擎 |

温度计算结果同时写入 `market_sentiment` 表缓存。页面读取路径：
1. 数据库最新记录 → 2. 实时调用东方财富 → 3. 随机 mock 降级

### 能力展现方式
- 大号渐变色卡片（紫/粉/橙/灰 对应 ICE/WARM/HIGH/RETREAT）
- 四指标横向排列的数字网格
- 板块列表（名称 + 百分比 + 领涨股）

### AI-Native 改造方向
- **自适应阈值**：当前 ICE/WARM/HIGH/RETREAT 的判定阈值（炸板率 > 0.4 = 冰点、晋级率 > 0.4 = 高潮……）是**硬编码**的。用历史数据训练 XGBoost 分类器，让模型学习最优阈值组合
- **情绪拐点预测**：不只告诉你「现在是高潮期」，而是预测「明天 65% 概率进入退潮期」。输入当日四指标 + 近 5 日趋势，输出次日状态概率分布
- **板块轮动分析**：当前只展示 Top 5 板块，AI 可以发现「AI 算力连续 3 天资金流入但涨幅不大 = 蓄力期」这类隐藏信号

## 页面三：自选股 `/watchlist`

### 当前能力
- **自选股列表**：代码 / 名称 / 现价 / 涨跌幅 / 匹配模式 / 操作按钮
- **观察池**：Tab 切换，用于区分「持有」和「跟踪中」
- **添加/删除/移池**：CRUD 操作

### 数据来源
| 数据 | 来源 | 逻辑 |
|------|------|------|
| 全部数据 | **内存 mock** (`user.py`) | 3 条写死的数据，无数据库读写 |

**这是整个项目中最薄弱的一环。** 自选股路由完全使用内存 mock，不读写数据库，不拉取真实行情。

### AI-Native 改造方向
- **智能排序**：不只按代码排序，而是按「距离止损位最近」「即将突破」「模式信号强度」等维度排列
- **异动提醒**：自选股出现放量、突破关键位置、触发模式信号时，AI 主动推送一段分析
- **组合诊断**：AI 分析你的自选股列表，"你的持仓过度集中在新能源，建议关注消费板块做对冲"

## 页面四：个股详情 `/stock/:code`

### 当前能力

**4.1 基本信息卡片**
- 股票名 / 代码 / 行业 / 市场状态徽标
- 现价 + 涨跌幅（红涨绿跌）
- 成交量 / 换手率 / 板块强度

**4.2 经典模式匹配**
- 展示该股是否触发「平台突破放量」或「分歧转一致」
- 每个模式卡片包含：模式名、描述、确认条件、失败条件、风控参考价、回测统计

**4.3 K线图**
- lightweight-charts 渲染，红涨绿跌蜡烛图
- 可拖动的入场/止损/止盈三条价格线（RiskRewardPanel）
- 实时计算盈亏比

### 数据来源

| 数据 | 来源 | 逻辑 |
|------|------|------|
| 基本信息 | `daily_quotes` 表 → AkShare 新浪 K线 → mock | 三级降级 |
| 板块强度 | **随机数** | `random.uniform(30, 95)` |
| 模式匹配结果 | `pattern_scan.py` 规则引擎 | 真实 DB 数据输入 + 硬编码规则判断 |
| 60 日最高价 | 数据库 `daily_quotes` 近 60 条 | 真实计算 |
| 20 日均量 | 数据库 `daily_quotes` 近 20 条 | 真实计算 |
| 是否涨停 | 东方财富涨停池 `getTopicZTPool` | 实时 HTTP 查询 |
| 主力净流入 | AkShare `stock_individual_fund_flow` (新浪) | 列 3 元→万元 |
| 市场状态 | 数据库 `market_sentiment` 最新记录 | 影响模式是否启用 |
| 回测统计 | `pattern_backtest_cache` 表 | 120 天滑动窗口回测结果 |
| K线数据 | 数据库 `daily_quotes` → AkShare 新浪 → mock | 三级降级 |

### 评估：每个数据点的真实度

```
✅ 真实数据：close, high, low, volume, turnover, 60日高, 20日均量, 是否涨停, 主力净流入
⚠️ 规则硬编码：模式匹配的阈值 (2%, 1.5倍, 3000万)
❌ 随机mock：板块强度 (sector_strength)
```

### AI-Native 改造方向

**4.1 模式匹配 → AI 形态识别**
- 当前：规则引擎（`if close near 60-day-high AND volume > 1.5x avg → 突破`）
- AI 化：将历史 K 线图转为图像，用 CNN/ViT 识别技术形态（W 底、头肩顶、旗形整理……），不限于 2 种硬编码模式
- 或者更轻量：将 OHLCV 序列输入 LSTM/Transformer，直接输出「未来 3 日上涨概率」，不再依赖"模式→回测→胜率"的间接路径

**4.2 板块强度 → 真实数据**
- 当前：`random.uniform(30, 95)`
- 修复：从 `stock_sector_spot` 中匹配该股所属板块的涨跌幅

**4.3 K线图 → AI 叠加层**
- 在 K 线图上叠加 AI 生成的支撑/压力位（不止是固定百分比止损）
- 标注历史相似形态位置："2025年3月和2024年11月出现过类似的量价结构，后续走势分别为……"

**4.4 AI 个股诊断**
- 点击「AI 诊断」按钮
- 输入：近 60 天 K 线 + 模式信号 + 市场温度 + 板块走向
- 输出：一段综合评估文字（不是"买/卖"，而是"该股处于什么阶段，历史上类似情况下胜率多少，当前风险点在哪里"）

## 页面五：复盘简报 `/review`

### 当前能力
- **市场状态徽标**：ICE/WARM/HIGH/RETREAT 彩色圆角标签
- **四项指标**：连板高度 / 晋级率 / 炸板率 / 昨涨停均收
- **板块涨幅排行**：同大盘页的 Top 5 板块
- **模式信号**：当日涨停股前 5 只的模式匹配结果
- **风险预警**：基于炸板率 / 连板高度 / 晋级率的动态风险提示

### 数据来源

| 数据 | 来源 | 逻辑 |
|------|------|------|
| 市场温度 | `market_sentiment` 表 | 与大盘页同一套数据，`calculate()` 计算状态 |
| 资金流向 | AkShare 新浪源 (`stock_market_fund_flow` + `stock_sector_spot`) | 实时拉取 |
| 模式信号 | 数据库 `limit_up_records` + `daily_quotes` | 取涨停股 → 前 5 只 → 扫描与 review.py 中硬编码条件匹配 |
| 风险预警 | 规则引擎 | 炸板率 > 35% / 连板高度 ≤ 2 / 晋级率 < 15% |

### 评估

```
✅ 真实数据：市场四指标、板块涨幅、涨停股列表
⚠️ 规则硬编码：模式信号的判断条件在 review.py 中又写了一遍 (与 pattern_scan.py 重复)
```

### AI-Native 改造方向

**这是整个项目中 AI 化 ROI 最高的页面。**

**5.1 AI 复盘报告生成**
- 当前：页面展示 4 个数字 + 板块排行 + 模式信号列表
- AI 化：后端调用 LLM，将当天的所有数据灌入 prompt，生成一段 **500 字的盘后总结**：
  - 今日市场发生了什么（数据陈述）
  - 为什么值得关注（因果关系）
  - 明天应该看什么（前瞻建议）

这不是简单地把数字翻译成中文，而是加入：
- 历史比较："今天的炸板率 32%，过去三个月只有 5 天超过这个水平，之后 3 天平均跌幅 2.3%"
- 模式联动："平台突破模式今日触发 3 次，但回测显示当前市场状态下（高潮期）该模式胜率仅 48%，建议谨慎"

**5.2 自适应风险预警**
- 当前：写死的三个阈值（炸板率 > 0.35、连板 ≤ 2、晋级率 < 0.15）
- AI 化：用历史数据学习「什么条件下次日下跌概率显著升高」，输出动态风险评分

## 现有 AI 相关功能盘点

### 回测引擎 (`backtest.py`)
- **作用**：对模式 1（平台突破）和模式 2（分歧转一致）做 120 天滑动窗口回测
- **方法**：遍历所有股票的每日数据，模拟入场 → 跟踪止损/止盈/到期退出 → 统计胜率和盈亏比
- **与 AI 的关系**：回测是**监督学习的基础设施**。当前用规则筛选样本，如果改为用 ML 模型预测，这套回测框架可以直接拿来评估模型效果
- **当前局限**：止损 3%、止盈 5%、持仓 3 天全部写死，不同模式无法自适应

### 模式扫描引擎 (`pattern_scan.py`)
- **作用**：`PATTERN_REGISTRY` 定义模式 → 判断函数逐条检查 → `scan_stock()` 统一入口
- **与 AI 的关系**：这个架构可以直接接入 ML。把 `check_breakout_volume()` 替换为 `model.predict(features)`，其余流程不变
- **当前局限**：只有 2 个模式，函数签名写死 8 个参数

## AI-Native 改造优先级与实施路径

### 第一阶段：AI 增强现有页面（本周可做）

| 优先级 | 功能 | 技术方案 | 依赖 |
|--------|------|---------|------|
| P0 | AI 复盘报告 | `POST /api/v1/review/summary` → LLM 生成 500 字 | 需要 LLM API Key |
| P1 | 自然语言搜索 | 搜索框输入 → LLM 解析意图 → 路由到已有 API | 同上 |
| P2 | 修复板块强度 mock | 从板块数据匹配，去掉 `random.uniform` | 无 |
| P2 | 修复自选股 mock | 接入数据库 + 真实行情 | 无 |

### 第二阶段：AI 嵌入决策链路（本月可做）

| 优先级 | 功能 | 技术方案 | 依赖 |
|--------|------|---------|------|
| P1 | 个股 AI 诊断按钮 | 灌入 K 线 + 模式 + 温度 → LLM 输出评估 | LLM API |
| P1 | ATR 动态止损 | 替换写死的 3%/5% | 无外部依赖 |
| P2 | AI 自选排序 | 按信号强弱而非字母排序 | 无 |
| P2 | 情绪拐点预测 | XGBoost 分类器，输入五日趋势 → 输出次日状态概率 | 需积累 ≥ 6 个月数据 |

### 第三阶段：模型驱动（需数据积累，远期）

| 功能 | 技术方案 | 门槛 |
|------|---------|------|
| ML 形态识别 | 历史 K 线打标签 → CNN/LSTM 训练 → 输出模式概率 | 需要标注数据 |
| 相似形态检索 | K 线序列 embedding → FAISS 向量检索 | 中等 |
| AI Agent 盘中监控 | 实时数据 → 触发条件 → 微信推送确认 | 需要盘中数据源 + 合规考量 |
