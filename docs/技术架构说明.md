# Vates - 技术架构说明

## 1. 整体架构

```
┌─────────────────────────────────────────────────────┐
│                     客户端层                          │
│  Vue 3 + TypeScript + Vite                           │
│  TradingView Lightweight Charts                      │
│  Pinia (状态管理)                                     │
└──────────────────┬──────────────────────────────────┘
                   │ HTTP REST + JSON
                   ▼
┌─────────────────────────────────────────────────────┐
│                     服务层                            │
│  FastAPI (Python 3.11+)                              │
│  ├─ routers/   API 路由                              │
│  ├─ services/  业务逻辑                              │
│  └─ schemas/   请求/响应模型                          │
└────────┬──────────────────────┬──────────────────────┘
         │                      │
         ▼                      ▼
┌─────────────────┐   ┌─────────────────┐
│   PostgreSQL 15  │   │    Redis 7       │
│   主数据存储      │   │   缓存 + 队列     │
└─────────────────┘   └─────────────────┘
         ▲
         │
┌─────────────────┐   ┌─────────────────┐
│  Celery Worker  │   │   AkShare        │
│  定时任务调度    │──▶│   数据抓取        │
└─────────────────┘   └─────────────────┘
```

## 2. 技术选型

| 层 | 技术 | 版本 | 选型理由 |
|----|------|------|----------|
| 前端框架 | Vue 3 + TypeScript | 3.4+ | 轻量、Composition API、`<script setup>` 简洁 |
| 构建工具 | Vite | 5.x | 快速 HMR，Vue 生态一等支持 |
| UI 组件库 | 无（手写样式） | - | 极致轻量，完全控制 |
| 图表库 | TradingView Lightweight Charts | 4.x | 专业金融图表，支持拖拽标注，框架无关 |
| 状态管理 | Pinia | 2.x | Vue 3 官方推荐，类型友好 |
| 路由 | Vue Router | 4.x | Vue 3 官方路由 |
| HTTP 客户端 | Axios | 1.x | 拦截器、请求取消 |
| 后端框架 | FastAPI | 0.110+ | 异步支持、自动文档、类型校验 |
| ORM | SQLAlchemy | 2.x | 异步支持成熟 |
| 数据校验 | Pydantic | 2.x | 与 FastAPI 深度集成 |
| 任务队列 | Celery + Redis | 5.x / 7.x | Python 生态标准方案 |
| 数据库 | PostgreSQL | 15 | 支持 JSONB、窗口函数 |
| 缓存 | Redis | 7 | 缓存热点数据、任务队列 |
| 数据源 | AkShare | latest | 免费开源 A 股数据 |
| 容器化 | Docker Compose | 3.8+ | 一键部署 |

## 3. 关键技术决策

### 3.1 前后端分离
- 前端：开发端口 5173，生产构建后由 Nginx 托管
- 后端：端口 8000，`/api/v1/*` 为业务 API，`/docs` 为自动生成的 Swagger 文档
- 开发阶段前端通过 Vite proxy 转发 `/api` 到后端

### 3.2 统一响应格式
```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```
错误码：`0` 成功, `1001` 参数错误, `1002` 资源不存在, `1003` 未登录, `1004` 权限不足, `2001` 数据源不可用, `3001` 服务器错误

### 3.3 数据抓取策略
- 仅盘后扫描（15:40 触发），取消盘中扫描
- Celery Beat 负责定时调度
- 数据抓取失败重试 3 次，仍失败标记 incomplete

### 3.4 认证方案
- MVP：固定用户 ID=1，JWT token 本地签发
- v1.5：微信 OAuth 2.0 + openid 绑定
- Access token 2h 有效，无 refresh token（MVP 简化）

### 3.5 前端路由
```
/                  首页（市场温度计 + 搜索）
/stock/:code       个股详情页
/watchlist         自选与观察池
/review            复盘简报（默认最新交易日）
/review/:date      指定日期复盘
```

### 3.6 部署策略
- **开发阶段**：本地开发，后端 `uvicorn --reload` 端口 8000，前端 `vite dev` 端口 5173，Vite proxy 转发 API
- **生产部署（后续）**：Docker Compose 容器化，Nginx 托管前端静态资源 + 反向代理后端
- 数据库与 Redis 本地安装或 Docker 运行
