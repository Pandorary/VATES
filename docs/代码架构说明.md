## 2. 后端分层架构

```
路由层 (routers/)    ← 接收 HTTP 请求，参数校验，调用服务层
   │
服务层 (services/)   ← 核心业务逻辑，算法实现，与数据层交互
   │
模型层 (models/)     ← ORM 模型，数据库表映射
   │
数据库 (PostgreSQL)
```

**调用规则**：
- 路由层不直接操作数据库，必须通过服务层
- 服务层通过 SQLAlchemy async session 操作模型
- 模型层仅定义表结构，不含业务逻辑
- Schemas 定义 API 契约（请求/响应格式）

## 3. 前端数据流 (Vue 3)

```
Views (页面)
  │ 调用 composables 获取数据
  ▼
Composables (组合式函数)
  │ 调用 services 发请求，管理 loading/error 状态
  ▼
Services (API 层)
  │ axios 封装，拦截器，错误处理
  ▼
Backend API

全局状态 (Pinia Store)
  ├─ useAppStore: 免责声明、用户信息
  └─ useWatchlistStore: 自选股缓存
```

## 4. 关键模块依赖关系

```
data_fetcher.py  ← 无依赖（仅依赖 AkShare）
market_temp.py   ← 依赖 models/market.py, models/stock.py
pattern_scan.py  ← 依赖 models/pattern.py, models/stock.py, market_temp.py (获取市场状态)
backtest.py      ← 依赖 models/pattern.py
review.py        ← 依赖 models/pattern.py, models/market.py, models/user.py
daily_jobs.py    ← 依赖 data_fetcher, market_temp, pattern_scan, review
```

## 5. 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件名 | snake_case | `market_temp.py` |
| 类名 | PascalCase | `MarketSentiment` |
| 函数/变量 | snake_case | `calculate_market_temp()` |
| API 路由 | kebab-case URL | `/api/v1/market/temperature` |
| Vue 组件 | PascalCase | `HomeView.vue` |
| Vue 组合式函数 | camelCase, use 前缀 | `useMarketTemp` |
| Pinia Store | camelCase, use 前缀 | `useAppStore` |
| 数据库表 | snake_case 复数 | `daily_quotes` |
| 数据库列 | snake_case | `trade_date` |
