> 为什么我本地项目文件夹上没有显示git跟踪的小绿标

  (thinking... 49b908e8-f14... ctrl+o to expand)

● Windows 上文件夹上的 Git 小绿标（图标叠加层）不是 Git 自带的功能，而是由第三方工具提供的。常见原因：
  
  **1. 没有安装 Git GUI 客户端**
  
  Git 自带的命令行不会在文件资源管理器中显示状态图标。需要安装以下任一工具：
  - **TortoiseGit**（最常用）
  - **GitHub Desktop**
  - **SourceTree**
  - **Git Extensions**
  
  **2. Windows 图标叠加层限制**
  
  Windows 最多支持 15 个图标叠加层，系统自带（OneDrive、SkyDrive 等）已经占了不少。如果安装过多个云同步工具，TortoiseGit 的注册表项可能被挤到后面，导致不显示。
  
  在注册表 `HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\ShellIconOverlayIdentifiers` 中可以看到所有叠加层，Windows 只认前 15 个。
  
  **解决方案**：
  - 安装 **TortoiseGit**（与 TortoiseSVN 同团队）
  - 如果安装后仍不显示，可以在上述注册表路径中将 TortoiseGit 的项名前面加空格（如 `  TortoiseGit...`）使其排在前面
  - 重启资源管理器或重启电脑后生效

> 我不需要onedrive的，是不是可以直接删掉？

  (thinking... 37a6e228-409... ctrl+o to expand)

● 可以。OneDrive 的图标叠加层占用了注册表位置，删掉就能给 Git 工具腾出空间。
  
  **步骤：**
  
  1. `Win + R`，输入 `regedit`，回车
  2. 导航到：`HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\ShellIconOverlayIdentifiers`
  3. 把以 `OneDrive` 开头的项都删除（通常有 3-5 个）
  4. 重启资源管理器（任务管理器 → 找到"Windows 资源管理器"→ 重启）
  
  **注意**：这只是移除 OneDrive 文件/文件夹上的状态图标，不影响 OneDrive 同步功能本身。
  
  删除后安装 TortoiseGit，它的叠加层图标就能正常显示了。
