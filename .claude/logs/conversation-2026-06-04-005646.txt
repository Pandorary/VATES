● ## 当前股票数据获取方式
  
  系统经历了一次架构迁移，从**定时预取**模式转向了**按需获取 + LLM 中心化**模式。
  
  ### 当前数据源
  
  | 数据源 | 获取方式 | 提供的数据 |
  |--------|---------|-----------|
  | 东方财富 (`push2.eastmoney.com`) | httpx HTTP 请求 | 实时行情（开/高/低/收/量/涨跌幅） |
  | 新浪财经 (`hq.sinajs.cn`) | httpx HTTP 请求 | 实时行情（交叉验证用） |
  | DeepSeek LLM | OpenAI 兼容 API | 股价回退数据、行业数据、搜索分类、分析预测 |
  
  ### 数据流架构
  
  ```
  用户请求 → FastAPI Router
    ├→ data_engine.py（预测端点）
    │    ├→ 东方财富 API ──→ 原始文本
    │    ├→ 新浪财经 API ──→ 原始文本
    │    └→ LLM ──→ 结构化提取 / 回退数据
    │         └→ 交叉验证（5% 容差） → 置信度标签
    ├→ llm.py（价格、分析、分类）
    │    └→ DeepSeek API
    └→ SQLite（读历史、写缓存）
         ├→ price_cache（按天缓存价格）
         ├→ ai_chat_cache / prediction_cache / industry_cache（按天缓存）
         └→ daily_quotes（只读，已不再写入）
  ```
  
  ### 关键特征
  
  - **纯按需获取**：没有定时任务，所有数据在用户请求时才拉取
  - **LLM 作为数据源**：东方财富/新浪获取不到的数据，由 LLM 直接"生成"
  - **多层缓存**：`price_cache`、`chat_cache`、`prediction_cache` 等均为按天缓存，同一天内重复请求命中缓存
  - **已移除的组件**：AkShare、Celery、Redis、APScheduler 等定时同步机制均已删除，`tasks/` 目录现在只剩空的 `__init__.py`
  
  ### 潜在问题
  
  - `daily_quotes`、`money_flow`、`limit_up_records`、`market_sentiment` 等表仍存在于模型定义中，但**已无数据写入机制**
  - 东方财富/新浪 API 无官方保障，可能随时变动或限流
  - LLM 生成的行情数据存在幻觉风险
