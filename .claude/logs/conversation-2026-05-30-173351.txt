● 分析结果

  data_summary 全部为 null、confidence 为
  "低"，说明数据引擎三个数据源全部未能提取到有效数据，最终触发了兜底逻辑。

  代码执行路径

  fetch_stock_data()
    ├─ _fetch_stock_raw() → 3个通道获取原始数据
    │    ├─ 东方财富 API: push2.eastmoney.com
    │    ├─ 新浪财经 API: hq.sinajs.cn
    │    └─ LLM 搜索（前两个失败才启用）
    │
    ├─ 对每个通道的原始数据，LLM 提取结构化 JSON
    │    └─ _parse_json_from_llm() 解析 → 如果失败，该通道作废
    │
    ├─ results 为空 → _llm_fallback_stock() 兜底
    │    └─ 直接问 LLM 要数据，但 DeepSeek 没有实时行情数据
    │    └─ 返回 {} 或解析失败 → confidence = "低"
    │
    └─ 返回给 prediction.py → code=400 拒绝

  最可能的原因

  1. 今天（2026-05-30）是周六，A股休市

  东方财富和新浪 API 在非交易日可能返回空数据或格式变化的数据，导致 LLM
  无法从中提取出有效字段。

  2. 东方财富 API 返回格式变化

  _fetch_stock_raw 中东方财富 API
  的字段是硬编码的（f43,f44...），如果接口格式变动，返回的 JSON
  结构可能改变，LLM 提取时解析失败。

  3. 两个 API 都不可达

  网络/防火墙问题导致 HTTP 请求失败（异常被 catch 后只打
  warning），所有通道都静默失败。

  先确认 API 是否通达，在浏览器直接访问：

  https://push2.eastmoney.com/api/qt/stock/get?secid=1.600519&fields=f43,f44,f45
  ,f46,f47,f48,f50,f51,f52,f55,f57,f58,f170,f171
  https://hq.sinajs.cn/list=sh600519

  看看能否正常返回数据。如果浏览器能返回、但后端获取不到，则是网络/超时问题。如
