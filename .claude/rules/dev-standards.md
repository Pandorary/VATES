# 开发规范

## 按钮统一铁律（最高优先级）

所有按钮（主/次/默认/禁用/弹窗/列表操作/图标按钮）必须：
- **高度**：统一 40px（小屏 36px，大屏 44px，仅此两档）
- **内边距**：水平 16px～24px，垂直 0
- **圆角**：统一 8px
- **边框**：统一 1px
- **阴影**：默认无阴影，悬浮/聚焦按规范加
- **字体**：统一 14px/500
- **状态**：hover/active/disabled 样式全项目一致

> 例外：图标按钮尺寸 32px/40px，需在注释中说明原因。

## 配色强制执行（引用 design token）

- 主色：`#3B82F6`（禁止魔改）
- 辅助色/危险色/成功色按 shadcn/ui 默认语义色
- 禁止出现：硬编码 `#000000` / `#FFFFFF` / 任意未定义色值
- 按钮文字色对比度 ≥ 4.5:1

## 全局组件统一校验

使用 shadcn/ui + Radix 组件库，位于 `frontend/src/components/ui/`：
- `button`, `input`, `switch`, `select`, `checkbox`, `dialog`, `badge`, `card`, `table`, `separator`, `tooltip`, `textarea`
- **默认圆角** = 8px
- **默认间距** = 4 的倍数（4/8/12/16/24/32px）
- **禁止**局部覆写样式类名或内联 style（使用 Tailwind 类名）

## 文字排版强制标准

- **字号层级**：12/14/16/20/24px
- **行高**：1.4～1.6（标题 1.3）
- **字间距**：默认 normal，标题可 +0.2px
- **段落边距**：下边距 16px

## 交互强制符合直觉

- **点击热区** ≥ 44px（移动端）/ ≥ 32px（PC）
- **弹窗**：ESC 关闭、点击遮罩关闭（破坏性操作除外）
- **状态反馈**：点击后 200ms 内给出视觉/加载反馈
- **禁止**：无提示跳转、双重滚动条、focus 丢失

## 执行流程（每次修改前必须遵守）

1. 先全局巡检所有按钮及通用组件
2. 列出样式/配色/尺寸不统一的问题清单
3. 统一制定通用样式规范（如有缺失，沿用 shadcn/ui 默认）
4. 输出统一化样式代码（Tailwind 类名或 CSS 变量）
5. 最后做交互优化

## 编码纪律（强制执行）

### 1. 读后再改

**每次 Edit 前必须先 Read 目标文件的当前内容**，禁止凭记忆或缓存内容盲改。
- 文件被多次编辑后内容已变，用旧内容匹配必然失败
- 如果 Edit 匹配失败，第一反应是 Read 重新获取内容，而不是反复试不同字符串

### 2. 改完即验

**每次修改后立即验证**，不要攒一堆改完再验：
- 改后端代码：curl 测试 API 响应
- 改前端代码：检查浏览器控制台是否报错
- 改多个文件：逐个验证，确认一个没问题再改下一个

### 3. 同一文件多次编辑后检查整体结构

**对同一文件进行 3 次以上编辑后，必须完整读取并检查**：
- 括号是否匹配
- 函数是否重复定义
- import 是否遗漏
- 代码逻辑是否自洽

### 4. API 路由：静态路径在动态参数之前

**FastAPI 路由按注册顺序匹配**，静态路径必须放在动态参数路径之前：
```
GET /holdings/total-assets    （先注册）
PUT /holdings/total-assets    （先注册）
GET /holdings/{holding_id}    （后注册）
```
否则 `total-assets` 会被当作 `holding_id` 解析，导致 422。

### 5. 服务重启：先杀旧进程

**重启后端/前端前，先确认旧进程已终止**：
- 检查端口是否释放：`netstat -ano | grep <port>`
- 杀掉占用进程后再启动
- 启动后验证端口是否正确监听

## 特别说明

若某个场景确实无法遵守规范，必须在代码注释中写明原因、影响范围、临时方案，并标注 `// FIXME: 规范例外，待整改`。
